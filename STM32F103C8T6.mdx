一、 MCU 内存结构与变量存储详解
ROM（FLSH）：只读寄存器，断电不丢失，专门放程序代码和常量
RAM：读写仓库，断电丢数据，存放堆、栈、全局变量

ROM可以将64k大小设为63kb留1kb存储用户信息
RAM无特殊需求不动，除非留一段给DMA、USB等

对于内存大小的修改不看能不能，要看为什么改。

微库：可以减少FLSH内存的使用，不建议使用malloc（没有碎片整理）
标准C库：可重入，可以使用浮点，支持文件读写

提示：可以以下代码代替浮点打印
float v = 3.1415；
printf("%d.%02d", (int v), (int)(v*100)%100)
遇见负数会出问题，同时两位小数不是四舍五入而是直接舍弃

.bss和.data之间没有隔离，栈溢出会覆盖全局变量
如果.data段很大会拖慢启动时间
1.全局变量
初始化的存入.data段，未初始化的存入.bss段
程序烧录时.data数据会存入FLSH只读区域里，系统启动时会被复制到RAM里
.bss数据不占有FLSH，启动时启动代码会将对应的RAM区域清零

2.局部变量
局部变量会存放在栈空间中，函数返回时会自动释放
静态局部变量会放在.bss/.data段，不占用栈空间，即使没执行到初始化的位置但仍旧分配了内存，初始化之前值为0

3.extern修饰词
extend修饰的变量是从其他文件中的全局变量引用来的
一般可在变量声明的头文件中使用extern修饰词修饰，是用谁调用

4.static修饰词
static修饰的变量是静态变量，程序运行期间只初始化一次
静态局部变量的生命周期是程序运行期间
使用场景：大缓冲区，只在本文件中使用的变量（防止命名冲突），需要记录上次值的

5.const关键字
修饰全局变量时变量会放入FLSH中
修饰局部变量时依旧放在栈空间中
编译器会阻止你去修改这个变量

6.volatile关键字
修饰变量时可以防止编译器优化，保证每次都从内存中读取，而不是在寄存器中读取
不能保证线程安全，只能解决编译器优化，不能保证原子性和互斥性
应用场景：内存映射I/O寄存器、中断标志位、setjmp/longjmp 场景

编译器优化：当编译器发现某个变量被多次读取且没有显示修改时会把值缓存到寄存器中，加速数值读取
setjmp(env)：把当前程序状态（寄存器、栈指针、程序计数器等）保存到 env 中，直接调用时返回 0。
longjmp(env, val)：恢复 env 保存的状态，程序会重新从对应的 setjmp 调用处返回，但这次返回值是 val（非零）。

RAM内存分布：
栈（stack）在顶部向下生长 推荐设为4kb，默认为1kb，对 MQTT/HTTP/JSON 等协议栈项目，分配 8KB 栈完全合理 起始地址：0x20000000
堆（Heap） 在.bss段结束位置上面向上生长 默认512b，可以防止隐式使用（一般裸机不使用），在FreeRTOS中使用堆分配任务，队列
全局/静态变量

快速记忆：全局在低，堆往上走，栈在头顶，往下开口
在资源允许的情况下，栈宁大勿小，堆按需分配不用可关

栈溢出检测方法：
1.水印法：可以在栈底写一个水印看是否被覆盖（简单，适用于裸机开发）
2.FreeRTOS中：FreeRTOS 提供了两种任务栈溢出检测机制，通过配置 configCHECK_FOR_STACK_OVERFLOW 宏来启用。

    // 0 = 关闭检测

    #define configCHECK_FOR_STACK_OVERFLOW 1

启动流程
上电 -> 取栈顶与复位向量 -> 启动文件初始化.data/.bss -> SystemInt -> main

系统时钟
时钟源：HSI（内部高速时钟）、HSE（外部高速时钟）、LSI（内部低速时钟）、LSE（外部低速时钟）、PLL（锁相环）
HSL（内部高速时钟）、HSE（外部高速时钟）、PLL（锁相环）主要给CPU和主要外设使用
低速时钟（给看门狗、RTC 等用）
PLL	Phase Locked Loop（锁相环）	不是一个独立振荡器，而是把 HSI/HSE 的时钟倍频到更高频率（如 72MHz、168MHz），再提供给系统时钟
系统时钟经过AHB/APB分频给外设

GPIO
八种工作模式：
1.上拉输入：主要用于按键输入
2.下拉输入：主要用于检测高电平有效信号
3.浮空输入：通常用于串口输入和中断信号检测
4.模拟输入：主要用于ADC模数转换
5.推挽输出：主要用于高速信号传输（SPI），强驱动
6.开漏输出：主要用于IIC等，需要接上拉电阻
7.复用推挽输出：片选设配输出，硬件SPI，USART，PWM
8.复用开漏输出：硬件IIC
由硬件决定的用复用输出

中断（NVIC）
抢占式优先级：决定中断嵌套
响应式优先级：决定同等抢占式优先级的响应顺序

定时器
用途：定时中断，输入捕获、PWM

串口
常用DMA加中断空闲，特别适合不定长数据帧

DMA
硬件数据搬运，外存和内设之间数据搬运、内存和内存之间搬运
功能：降低CPU占用率
