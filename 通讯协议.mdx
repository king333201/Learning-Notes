一、ADC
    先配置ADC通道，设置采样时间转化精度，启动转换读取结果
    如果传感器是模拟电压直接接ADC引脚就行

二、IIC
    同步通讯，一条时钟线（SCL），一条数据线（SDA） 半双工
    主机完全掌控SCL的控制权可使用推挽输出

    为避免线路短路SDA线上严禁任何设备输出高电平，采用开漏输出加上拉电阻的方式传输数据
    低电平下拉，高电平浮空通过外置的上拉电阻拉高电平（所以上拉为弱上拉）
    上拉电阻类似于弹簧，低电平为向下的拉力

    起始信号：SCL高电平期间，SDA从高电平切换到低电平

    终止信号：SCL高电平期间，SDA从低电平切换到高电平

    接收一个字节:SCL低电平期间，从机将数据位依次放到SDA线上(高位先行)，
    然后释放SCL，主机将在SCL高电平期间读取数据位所以SCL高电平期间SDA不允许有数据变化，
    依次循环上述过程8次，即可接收一个字节(主机在接收之前，需要释放SDA)和发送单元差不多
    SCL低电平:从机数据搭载到SDA上
    SCL高电平:主机读取SDA上的数据

    发送应答:主机在接收完一个字节之后，在下一个时钟发送一位数据，数据0表示应答，数据1表示非应答
    接收应答:主机在发送完一个字节之后，在下一个时钟接收一位数据，判断从机是否应答，
    数据0表示应答，数据1表示非应答(主机在接收之前，需要释放SDA)在字节传输完成后
    SCL低电平时:接收端SDA拉低/拉高，低电平代表应答
    SCL高电平时：发送端读取SDA，低电平代表对方回答了

    IIC时序基本单元分为：起始条件、终止条件、发送一个字节、接收一个字节、发送应答，接收应答
    IIC指定地址写时序：起始位 + 指定从机（写操作） + 应答位 + 指定地址 + 应答位 + 写入的数据 + 应答位 + 终止位
    IIC指定地址读时序：起始位 + 指定从机（写操作） + 应答位 + 指定地址 + 应答位 + 起始位 + 指定从机（读操作） + 应答位 + 读取的数据 + 应答位 + 终止位

三、SPI
    同步通讯，全双工
    四线：SCK（时钟）、MOSI（主机发从机收）、MISO（从机发主机收）、CS/NSS（片选，低电平有效）
    全双工：MOSI和MISO同时收发，主机发一个字节的同时也在收一个字节

    推挽输出（无上拉电阻），驱动能力强，速度远高于IIC（可达几十MHz）
    从机不能主动发起通信，必须等主机拉低CS才能干活

    四种工作模式（由CPOL和CPHA组合决定）：
        CPOL（时钟极性）：空闲时SCK电平，0=低电平，1=高电平
        CPHA（时钟相位）：0=第一个边沿采样，1=第二个边沿采样
        模式0（CPOL=0, CPHA=0）：空闲低，上升沿采样 ← 最常用
        模式1（CPOL=0, CPHA=1）：空闲低，下降沿采样
        模式2（CPOL=1, CPHA=0）：空闲高，下降沿采样
        模式3（CPOL=1, CPHA=1）：空闲高，上升沿采样

    多从机方案：
        1. 每个从机独立的CS引脚 — 常用，响应快
        2. 菊花链（Daisy Chain）— 一个从机输出接下一个从机输入，省引脚但延迟大

    SPI vs IIC 对比：
        | 特性       | SPI                    | IIC                    |
        |-----------|------------------------|------------------------|
        | 信号线     | 4线（含片选）            | 2线（SCL+SDA）          |
        | 通信方式   | 全双工                  | 半双工                  |
        | 速度       | 快（几十MHz）            | 慢（标准100k/快速400k/高速3.4M） |
        | 输出方式   | 推挽                    | 开漏+上拉               |
        | 寻址方式   | 硬件片选（CS）           | 软件地址（7位/10位）     |
        | 多从机     | 需额外CS引脚             | 一条总线挂多个，靠地址区分 |
        | 应答机制   | 无（主对从单向控制）      | 有ACK/NACK              |
        | 适用场景   | 高速、大数据量（Flash、屏幕、ADC） | 引脚受限、多设备低速通信（传感器、EEPROM） |

四、UART
    异步通讯，全双工，点对点（一对一）
    无时钟线，双方约定相同的波特率来同步

    数据帧格式：
        起始位（1位，低电平） + 数据位（5~9位，通常8位，LSB先发） + 校验位（可选，奇/偶校验） + 停止位（1/1.5/2位，高电平）
        空闲状态：总线保持高电平，起始位的低电平是“提醒对方我要发数据了”

    波特率：每秒传输的码元数（bps），常见9600、115200
    收发双方波特率误差需 < 2%~3%，否则累积误差会导致数据错位

    电平标准：
        TTL/CMOS：芯片直连，3.3V/5V，传输距离短（板级）
        RS232：±3~±15V，逻辑1=负电压，逻辑0=正电压，传输距离约15m
        RS485：差分信号（A/B线），抗干扰强，传输距离可达1200m，支持多机（一主多从）

    串口接收不定长数据：
        常用DMA + 空闲中断（IDLE）：
        1. 配置DMA从USART数据寄存器搬运到接收缓冲区
        2. 当总线空闲（一个字节时间内无新数据）时触发IDLE中断
        3. 在IDLE中断中计算已接收字节数，处理数据帧，重置DMA
        适合Modbus、自定义协议等不定长帧场景

五、CAN
    异步通讯，半双工，多主架构（任意节点可主动发数据）
    差分信号：CAN_H和CAN_L两根线，显性电平（逻辑0）压差约2V，隐性电平（逻辑1）压差≈0V
    显性覆盖隐性：多个节点同时发时，发显性的赢 — 这就是总线仲裁的基础

    物理层：CAN_H和CAN_L终端各接一个120Ω电阻，消除信号反射

    帧类型：
        数据帧：发送数据的帧，标准帧11位ID，扩展帧29位ID
        遥控帧：请求其他节点发送数据
        错误帧：检测到错误时发送
        过载帧：通知其他节点自己还没准备好

    数据帧结构（标准帧）：
        帧起始(1位) + 仲裁段(ID 11位 + RTR 1位) + 控制段(IDE + r0 + DLC 4位) + 数据段(0~8字节) + CRC段(15位+界定符) + ACK段(1位+界定符) + 帧结束(7位)

    仲裁机制：
        ID越小优先级越高。节点在发ID的同时也在监听总线，如果自己发隐性(1)但总线变成显性(0)，说明有更高优先级的节点在抢总线，自己主动退出发送，等下次重试。
        仲裁在ID段完成，不破坏数据，赢家继续发，输家静默。

    位时序（决定波特率）：
        一个位时间 = 同步段 + 传播段 + 相位缓冲段1 + 相位缓冲段2
        采样点一般设在相位缓冲段1结束处，推荐 75%~87.5%
        常用波特率：125k、250k、500k、1M

    过滤器：
        每个CAN控制器有若干过滤器组，可配置为：
        列表模式：只接收指定ID的报文
        掩码模式：按掩码匹配一组ID（掩码=0的位不关心，掩码=1的位必须匹配）
        用于过滤不需要的报文，减少CPU中断负担

    错误处理：
        5种错误：位错误、填充错误、CRC错误、格式错误、应答错误
        错误计数器：发送错误计数和接收错误计数，超过阈值节点自动进入错误被动或总线关闭状态

    CAN FD（CAN with Flexible Data-rate）：
        传统CAN数据段最多8字节，速率最高1Mbps
        CAN FD数据段速率可变（最高可达8Mbps），数据段最大64字节
        向下兼容传统CAN

    CAN vs 其他协议对比：
        | 特性       | CAN               | UART            | SPI              | IIC            |
        |-----------|-------------------|-----------------|------------------|----------------|
        | 拓扑       | 总线型（多主多从）   | 点对点           | 一主多从           | 一主多从         |
        | 信号线     | 2线差分             | 2线（TX+RX）     | 4线               | 2线             |
        | 全双工     | 半双工              | 全双工           | 全双工             | 半双工           |
        | 速度       | 125k~1M（FD: 8M）   | 一般≤1M          | 几十MHz            | 100k~3.4M       |
        | 帧长度     | 0~8字节（FD: 64字节）| 通常5~9位        | 任意长度            | 任意长度          |
        | 错误检测   | 强（CRC+ACK+位监控） | 弱（仅奇偶校验）   | 无                 | 弱（ACK）        |
        | 适用场景   | 汽车、工业控制       | 调试、简单通信     | 高速短距、Flash     | 板级低速多设备    |

六、MODBUS
    应用层协议，主从架构（一主多从），主机发起请求，从机响应
    物理层不限定，可跑在 RS232、RS485、TCP/IP 上
    设备地址：1~247，0=广播地址（所有从机执行但不回复），248~255 保留

    四种数据模型：
        | 类型           | 名称         | 大小   | 访问  | 地址范围      | 用途示例             |
        |---------------|-------------|--------|-------|--------------|---------------------|
        | 线圈（Coil）          | 开关量输出    | 1位    | 读写  | 00001~09999  | 继电器、LED、阀门     |
        | 离散输入（Discrete）  | 开关量输入    | 1位    | 只读  | 10001~19999  | 按钮、限位开关、传感器 |
        | 保持寄存器（Holding）  | 模拟量输出    | 16位   | 读写  | 40001~49999  | 设定值、参数配置       |
        | 输入寄存器（Input）    | 模拟量输入    | 16位   | 只读  | 30001~39999  | 温度、湿度、电压读数   |

    数据模型本质上只是地址编号，没有物理区别。同一个设备可能线圈和保持寄存器指向同一块内存。
    两个16位寄存器可拼成一个32位值（高字节在前），如浮点数、32位计数器。

（一）RTU 模式
    基于串口（RS232/RS485），二进制传输，每个字节8位，效率高，最常用

    RTU 帧格式：
        | 从机地址(1B) | 功能码(1B) | 数据区(N B) | CRC校验(2B, 低字节在前) |
    帧与帧之间至少 3.5 个字符时间（t3.5）的静默间隔作为帧结束标志
    字节间间隔不能超过 1.5 个字符时间（t1.5），否则视为帧异常

    波特率与字符时间对应（11位/字符，含起始位+8数据+1校验+1停止）：
        | 波特率  | 3.5字符时间(t3.5) | 1.5字符时间(t1.5) |
        |---------|-------------------|-------------------|
        | 9600    | ≈4.01ms           | ≈1.72ms           |
        | 19200   | ≈2.00ms           | ≈0.86ms           |
        | 115200  | ≈0.33ms           | ≈0.14ms           |

    CRC16 计算口诀：
        CRC初始值=0xFFFF，逐字节异或后右移，移出1则异或多项式0xA001，循环8次，两个字节都处理完后高低字节交换

（二）ASCII 模式
    基于串口，每个字节转为两个ASCII字符（0~9、A~F），可读性强但效率减半

    ASCII 帧格式：
        | 起始符':' | 从机地址(2字符) | 功能码(2字符) | 数据区(0~2x252字符) | LRC(2字符) | 结束符'\r\n' |
    帧头用冒号':'，帧尾用回车换行，无需依赖时间间隔判断帧边界

    LRC 校验：从地址到数据区最后一个字节累加，结果取反+1

（三）TCP 模式
    基于以太网（TCP端口502），速度快，支持多主机并发，去掉从机地址用MBAP头替代

    TCP 帧格式（ADU）：
        | 事务标识符(2B) | 协议标识符(2B, 固定0x0000) | 长度(2B) | 单元标识符(1B) | 功能码(1B) | 数据区(N B) |
        MBAP头(7字节) = 事务标识符 + 协议标识符 + 长度 + 单元标识符
        事务标识符：用于匹配请求与响应，每发送一帧自增
        单元标识符：串口转TCP网关时用于指定后端串口设备地址

    三种模式对比：
        | 特性       | RTU                | ASCII              | TCP                |
        |-----------|--------------------|--------------------|--------------------|
        | 物理层     | RS232/RS485         | RS232/RS485         | 以太网              |
        | 编码方式   | 二进制               | ASCII十六进制        | 二进制              |
        | 帧边界     | t3.5静默间隔         | 起始':' / 结束\r\n   | TCP包边界           |
        | 校验       | CRC16               | LRC                 | 无（由TCP保证）      |
        | 效率       | 高                   | 低（数据量翻倍）      | 最高                |
        | 应用场景   | 工业现场首选          | 调试/老旧设备         | 监控系统/远程访问    |

（四）常用功能码
    | 功能码 | 名称             | 操作对象       | 说明                     |
    |--------|-----------------|---------------|--------------------------|
    | 01     | 读线圈            | 线圈           | 批量读，最多2000个         |
    | 02     | 读离散输入         | 离散输入        | 批量读，最多2000个         |
    | 03     | 读保持寄存器       | 保持寄存器       | 批量读，最多125个寄存器     |
    | 04     | 读输入寄存器       | 输入寄存器       | 批量读，最多125个寄存器     |
    | 05     | 写单线圈           | 线圈           | 写单个，值=0xFF00(ON)或0x0000(OFF) |
    | 06     | 写单个寄存器       | 保持寄存器       | 写单个16位值              |
    | 15     | 写多个线圈         | 线圈           | 批量写，最多1968个         |
    | 16     | 写多个寄存器       | 保持寄存器       | 批量写，最多123个寄存器     |
    | 22     | 掩码写寄存器       | 保持寄存器       | AND掩码+OR掩码，原子操作    |
    | 23     | 读写多个寄存器     | 保持寄存器       | 一次请求同时读和写          |

    功能码 03（读保持寄存器）请求帧示例（RTU）：
        | 地址(0x01) | 功能码(0x03) | 起始地址高(0x00) | 起始地址低(0x00) | 寄存器数量高(0x00) | 寄存器数量低(0x03) | CRC低(0x05) | CRC高(0xCB) |
        含义：从地址1的设备，读保持寄存器地址0x0000开始的3个寄存器

    正常响应帧：
        | 地址(0x01) | 功能码(0x03) | 字节数(0x06) | 寄存器1高(0x00) | 寄存器1低(0x6B) | 寄存器2高(0x00) | 寄存器2低(0x13) | 寄存器3高(0x00) | 寄存器3低(0x00) | CRC低 | CRC高 |
        含义：寄存器0=107，寄存器1=19，寄存器2=0

    功能码 06（写单个寄存器）请求帧：
        | 地址(0x01) | 功能码(0x06) | 寄存器地址高(0x00) | 寄存器地址低(0x01) | 写入值高(0x00) | 写入值低(0x64) | CRC低 | CRC高 |
        含义：向地址1的设备，寄存器地址0x0001写入0x0064（100）

    正常响应帧与请求帧完全相同（原样回传）

（五）异常处理
    从机检测到错误时，响应帧功能码最高位（bit7）置1，后跟异常码

    异常响应帧格式：
        | 从机地址(1B) | 功能码+0x80(1B) | 异常码(1B) | CRC(2B) |

    常见异常码：
        | 异常码 | 名称                 | 触发条件                               |
        |--------|---------------------|---------------------------------------|
        | 01     | 非法功能码            | 功能码不支持                            |
        | 02     | 非法数据地址          | 起始地址+数量超出设备地址范围             |
        | 03     | 非法数据值            | 数据值不合法（如写线圈时值不是0xFF00/0x0000）|
        | 04     | 从站设备故障          | 设备内部错误，无法执行请求                |
        | 05     | 确认（ACK）           | 命令已接受但需要较长时间执行               |
        | 06     | 从站设备忙            | 设备正在处理上一个命令，请稍后重试          |

    示例：读不存在地址时产生异常
        请求：0x01 0x03 0x10 0x00 0x00 0x03 CRC —— 起始地址0x1000，但设备最大地址只有0x00FF
        异常响应：0x01 0x83 0x02 CRC —— 功能码0x03+0x80=0x83，异常码0x02=非法数据地址

（六）使用步骤

    1. 硬件连接
        RTU/ASCII：RS485时注意A/B线别接反，主机A接从机A，主机B接从机B
        每个设备A/B线并联，RS485总线两端各接一个120Ω终端电阻
        TCP：确认设备IP和端口502可达，网线直连或通过交换机

    2. 初始化配置
        // STM32 FreeRTOS 下 Modbus RTU 主站初始化示例
        // 配置串口：9600, 8N1（8位数据、无校验、1位停止）
        // 配置485方向控制引脚（DE/RE），发送前拉高，发送完拉低

        // 定时器配置（用于t3.5超时检测）
        // 波特率=9600时，t3.5≈4ms，超时定时器设5ms即可

    3. 发送请求与接收响应
        流程：组帧→CRC计算→开485发送→发送数据→关485发送→开485接收→等待响应→超时检测→CRC校验→解析

        主机发送读寄存器请求（伪代码逻辑）：
        a. 构造请求帧：addr + func(0x03) + 起始地址 + 寄存器数量
        b. 计算CRC16，追加到帧尾
        c. 拉高DE/RE，延时50μs等485切换
        d. 通过串口发送完整帧
        e. 拉低DE/RE，切换为接收模式
        f. 启动t3.5超时定时器，等待从机响应
        g. 串口中断接收数据，每次收到字节重置超时定时器
        h. 超时定时器触发 → 帧接收完成，校验CRC，解析数据

    4. 32位数据读取
        // 两个相邻保持寄存器拼成32位值
        // 寄存器N存高16位，寄存器N+1存低16位
        uint32_t value = (regs[0] << 16) | regs[1];

        // 浮点数（IEEE 754）：4字节，占两个寄存器
        // 寄存器0-1 → float，需要按设备手册确认字节序
        uint32_t raw = (regs[0] << 16) | regs[1];
        float value = *(float*)&raw;

    5. 断开连接
        RTU/ASCII：关闭定时器，关闭串口，释放RS485引脚
        TCP：关闭Socket连接

    | 现象                   | 可能原因                   | 排查方法                           |
    |-----------------------|---------------------------|-----------------------------------|
    | 无响应                 | 地址错误、波特率不匹配、A/B线接反 | 逐一排查：换地址试→示波器看波形→万用表量A/B电压差 |
    | 返回异常码0x02         | 起始地址或数量超出设备范围     | 查阅设备手册确认寄存器地址范围，减量重试           |
    | 返回异常码0x03         | 写入值不合法                | 检查写线圈时值是否为0xFF00/0x0000，寄存器值是否超范围 |
    | CRC校验失败            | 传输干扰、CRC算法错误        | 先用已知数据验证CRC计算函数，加终端电阻，降低波特率   |
    | 读到全是0或0xFFFF      | 寄存器未初始化、地址偏移错误   | 地址偏移1试试（有些设备地址从0开始，有的从1开始）    |
    | 只能读到前几个寄存器     | 请求数量超出设备限制         | 减到每次读10个以内，分多次读取                   |
    | 间歇性通信失败          | 总线终端电阻缺失、地线未接    | 485总线两端各加120Ω，确保所有设备共地            |
    | TCP连接不上            | 端口号错误、IP不在同网段      | ping测试→telnet IP 502→检查防火墙  |
    | 响应数据错位            | t3.5定时不准确             | 用逻辑分析仪抓波形验证时序，定时器中断优先级要够高    |

    调试技巧：
        - 用逻辑分析仪或USB转485接电脑，用Modbus Poll调试工具逐条验证
        - CRC计算脚本随手写一个，别每次都手算
        - 读寄存器时，先只读1个确认地址正确，再批量读
        - 上电后延时100ms再发命令，等从机初始化完成
        - RS485总线上设备数量多时，波特率不宜太高（≤19200 较稳）

    地址偏移陷阱：
        有些设备文档写“寄存器地址40001”，实际帧中起始地址是0x0000（40001-40001=0）
        有些设备文档写“寄存器地址0x0000”，实际对应40001
        规律：协议帧中的地址 = 设备文档地址 - 起始编号（40001、30001等）
        不确认时先读地址0x0000，看返回什么再判断
